   Value: <從PostgreSQL服務複製的Internal Database URL>

5. FRONTEND_URL
   Value: https://material-request-system.onrender.com
   注意: 部署完成後請更新為實際的服務 URL

   Value: trial@material-request.com

   Value: trial123456


═══════════════════════════════════════════════════════════════
  在 Render Dashboard 設定的步驟
═══════════════════════════════════════════════════════════════

1. 進入 Web Service: material-request-system
2. 點選左側「Environment」頁籤
3. 在「Environment Variables」區塊點選「Add Environment Variable」
4. 逐一新增上述 7 個環境變數
5. 每個變數新增後點選「Save」
6. 完成後，服務會自動重新部署

═══════════════════════════════════════════════════════════════
  如何取得 DATABASE_URL
═══════════════════════════════════════════════════════════════

1. 在 Render Dashboard 選擇 PostgreSQL 服務: material-request-db
2. 進入資料庫設定頁面
3. 找到「Connection」區塊
4. 複製「Internal Database URL」
5. 貼到 DATABASE_URL 環境變數的 Value 欄位

⚠️ 重要: 必須使用 Internal Database URL，不是 External URL
